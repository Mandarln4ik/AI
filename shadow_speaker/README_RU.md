# ShadowSpeaker - Локальный ИИ-ассистент для помощи в диалогах

## Возможности

✅ **Распознавание речи** с разделением спикеров (Faster-Whisper + pyannote)  
✅ **Поддержка 3 LLM провайдеров**: Ollama, LM Studio, llama.cpp  
✅ **Память диалога** - помнит контекст за последние 5 минут  
✅ **Визуальный контекст** - захват экрана для понимания ситуации в игре  
✅ **Прозрачный оверлей** поверх других приложений  
✅ **Импорт Whisper моделей** - поддержка .pt файлов (large-v3-turbo.pt и др.)  
✅ **GUI настроек** - удобный интерфейс выбора и загрузки моделей  

## Системные требования

- **ОС**: Windows 11
- **GPU**: NVIDIA с CUDA 12.4+ (поддерживает несколько карт)
- **Python**: 3.12+
- **RAM**: 8GB+ (рекомендуется 16GB)

## Установка

### 1. Установка зависимостей

```bash
cd shadow_speaker
pip install -r requirements.txt
```

### 2. Настройка аудио (для Discord/Teamspeak)

Для захвата звука из конкретных приложений используйте **VB-CABLE Virtual Audio Cable**:

1. Скачайте и установите: https://vb-audio.com/Cable/
2. В настройках Discord/Teamspeak установите выходное устройство на "CABLE Input"
3. В ShadowSpeaker выберите "CABLE Output" как источник аудио

### 3. Настройка LLM провайдера

#### Вариант A: Ollama
```bash
# Установите Ollama: https://ollama.ai
ollama pull llama3.2  # или phi3, mistral
ollama serve
```

#### Вариант B: LM Studio
1. Скачайте: https://lmstudio.ai
2. Загрузите модель через интерфейс
3. Включите "Local Server" в настройках (порт 1234)

#### Вариант C: llama.cpp (прямое использование .gguf)
```bash
pip install llama-cpp-python
# Укажите путь к .gguf файлу в настройках
```

### 4. Импорт Whisper модели (.pt файл)

Если у вас есть файл `large-v3-turbo.pt`:

1. Скопируйте файл в папку `shadow_speaker/models/`
2. Запустите приложение
3. Нажмите `Ctrl+S` для открытия настроек
4. Во вкладке "Whisper STT" укажите путь к файлу

Или отредактируйте `config.py`:
```python
whisper_model_path = "models/large-v3-turbo.pt"
```

## Запуск

```bash
python main.py
```

### Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `Ctrl+1/2/3` | Выбрать вариант ответа |
| `Ctrl+R` | Обновить варианты |
| `Ctrl+H` | Скрыть/показать оверлей |
| `Ctrl+S` | Открыть настройки |
| `Ctrl+Q` | Выход |

## Конфигурация

Основные настройки в `config.py`:

```python
# LLM провайдер
llm_provider = "ollama"  # или "lmstudio", "llama_cpp"
llm_model = "llama3.2"

# Whisper модель
whisper_model_name = "medium"  # или путь к .pt файлу
whisper_model_path = "models/large-v3-turbo.pt"

# Захват экрана
enable_screen_capture = True
screen_capture_interval = 10.0  # секунд

# Память диалога
max_context_age = 300  # 5 минут
```

## Архитектура

```
main.py              - Основное приложение
├── audio_capturer.py    - Захват аудио из приложений
├── speech_recognizer.py - STT + диаризация спикеров
├── llm_engine.py        - Генерация ответов (Ollama/LM Studio/llama.cpp)
├── screen_capture.py    - Захват экрана для визуального контекста
├── speaker_manager.py   - Управление профилями спикеров
├── overlay_ui.py        - Прозрачный оверлей с ответами
├── settings_gui.py      - GUI настроек
└── config.py            - Конфигурация
```

## Как это работает

1. **Захват аудио**: Приложение слушает звук из выбранного источника (микрофон или VB-CABLE)
2. **Распознавание**: Faster-Whisper транскрибирует речь, pyannote разделяет спикеров
3. **Контекст**: Последние 5 минут диалога сохраняются в памяти
4. **Визуальный анализ**: Каждые 10 секунд делается скриншот для понимания контекста игры
5. **Генерация**: LLM предлагает 3 варианта ответа на основе аудио + визуального контекста
6. **Оверлей**: Варианты отображаются поверх экрана, можно выбрать горячей клавишей

## Решение проблем

### LLM не отвечает
- Проверьте что Ollama/LM Studio запущены
- Убедитесь что модель загружена
- Проверьте порт в настройках

### Нет распознавания речи
- Проверьте выбор устройства ввода
- Убедитесь что Whisper модель загружена
- Для .pt файлов проверьте путь к файлу

### Оверлей не виден
- Попробуйте `Ctrl+H` для переключения видимости
- Проверьте прозрачность в настройках
- Запустите от имени администратора

### Проблемы с GPU
- Проверьте `torch.cuda.is_available()`
- Для нескольких карт настройте `CUDA_VISIBLE_DEVICES`
- Используйте `compute_type="float16"` для экономии памяти
