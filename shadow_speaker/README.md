# ShadowSpeaker - Локальный ИИ-ассистент для диалогов

## Описание
Приложение работает в фоне, перехватывает аудио из выбранных приложений (Discord, TeamSpeak и др.), распознает речь с разделением спикеров, генерирует варианты ответов и отображает их в прозрачном оверлее поверх экрана.

## Особенности
- **Локальная работа**: Все модели работают на вашем железе (2x NVIDIA: CMP 50HX + GTX 1630)
- **Разделение спикеров**: Автоматическое определение количества участников и кто из них "вы"
- **Захват аудио**: Выбор конкретного приложения для захвата звука (VB-Cable / Virtual Audio Cable)
- **Оверлей**: Прозрачное окно поверх всех программ с вариантами ответов
- **Конфиденциальность**: Никаких облачных запросов, всё локально
- **Гибкость LLM**: Поддержка Ollama, LM Studio и llama.cpp
- **Импорт Whisper**: Возможность использовать свои .pt модели (например large-v3-turbo.pt)
- **GUI настройки**: Встроенный интерфейс для управления моделями и параметрами

## Требования
- Windows 11
- NVIDIA драйверы (CUDA 12.x)
- Python 3.10+
- Virtual Audio Cable (для захвата звука из приложений)

## Установка
1. Установите Virtual Audio Cable и настройте маршрутизацию звука из Discord/Teamspeak
2. Установите один из LLM провайдеров:
   - **Ollama**: https://ollama.com → `ollama pull llama3.2` или `ollama pull phi3`
   - **LM Studio**: https://lmstudio.ai → загрузите модель через GUI, включите Local Server (порт 1234)
   - **llama.cpp**: pip install llama-cpp-python → укажите путь к .gguf файлу
3. Установите зависимости: `pip install -r requirements.txt`
4. Запустите: `python main.py`

## Использование
1. Откройте настройки (`Ctrl+S` или кнопка ⚙ в оверлее)
2. Выберите LLM провайдер (Ollama/LM Studio/llama.cpp)
3. Для Whisper: импортируйте вашу модель large-v3-turbo.pt через вкладку "Whisper STT" → "Обзор..."
4. Выберите приложение для захвата звука
5. Укажите ваш голосовой профиль (для идентификации "себя")
6. Начните разговор - оверлей появится автоматически при обнаружении речи
7. Используйте горячие клавиши для выбора варианта ответа

## Горячие клавиши
- `Ctrl+1/2/3`: Выбрать вариант ответа №1/2/3
- `Ctrl+R`: Обновить варианты
- `Ctrl+H`: Скрыть/показать оверлей
- `Ctrl+S`: Открыть настройки
- `Ctrl+Q`: Выход

## Настройки LLM провайдеров

### Ollama
- Хост по умолчанию: `http://localhost:11434`
- Команда для загрузки модели: `ollama pull <model_name>`
- Проверка: `curl http://localhost:11434/api/tags`

### LM Studio
- Хост по умолчанию: `http://localhost:1234`
- Модели загружаются через GUI LM Studio
- Включите "Local Server" в настройках LM Studio
- API совместим с OpenAI форматом

### llama.cpp
- Требует файл модели в формате `.gguf`
- Укажите полный путь к файлу в поле "Модель"
- Использует GPU автоматически при наличии CUDA

## Импорт Whisper модели
Если у вас есть готовая модель (например `large-v3-turbo.pt`):
1. Откройте настройки (`Ctrl+S`)
2. Перейдите во вкладку "Whisper STT"
3. Нажмите "Обзор..." и выберите ваш .pt файл
4. Сохраните настройки и перезапустите приложение

## Конфигурация для вашей системы
С 2 видеокартами NVIDIA (CMP 50HX + GTX 1630):
- Whisper будет использовать CUDA автоматически
- Для llama.cpp установите `n_gpu_layers=-1` для использования всех слоев GPU
- Рекомендуется использовать CMP 50HX для вычислений, GTX 1630 для дисплея
